  The dataset used for this project is the Online Retail dataset, which contains transactional records from an online retail store. This dataset includes information related to customer purchases and sales transactions. The attributes in the dataset include Invoice number, StockCode, product Description, Quantity purchased, InvoiceDate, Price of the product, Customer ID, and Country. Initially, the dataset consisted of 525,461 records and 8 attributes, making it a suitable dataset for advanced data mining and analytics tasks. The large number of records provides a strong foundation for meaningful analysis, while the variety of attributes allows exploration of relationships between customer behavior, product demand, and sales performance. Because the dataset includes both numerical and categorical variables, it is well suited for applying multiple data mining techniques such as predictive modeling, clustering, and association rule mining in later stages of the project.
    To begin the analysis, the dataset was loaded into Python using the Pandas library, which provides powerful tools for data manipulation and exploration. The structure of the dataset was inspected using functions such as head(), info(), and describe(). These functions helped identify the types of variables present, the number of records in each column, and summary statistics such as the mean, minimum, maximum, and standard deviation for numerical attributes. Through this initial inspection, it was observed that the dataset contained both categorical variables such as Invoice, StockCode, Description, and Country, as well as numerical variables such as Quantity and Price. The InvoiceDate variable was also recognized as a datetime feature, which could later be useful for analyzing time-based sales patterns.
    Data cleaning was an important step in preparing the dataset for further analysis. During the inspection process, missing values were identified in two columns. The Description column contained 2,928 missing values, and the Customer ID column contained 107,927 missing values. Missing values can reduce the accuracy of data analysis and predictive modeling, so appropriate steps were taken to address them. Rows with missing product descriptions were removed because the product description is an important feature for understanding the items being purchased. For the Customer ID column, missing values were filled with a default value so that the dataset remained consistent and complete. After performing these steps, the dataset was checked again to confirm that no missing values remained.
      Another important cleaning step involved identifying and removing duplicate records. Duplicate entries can occur due to system errors or repeated transaction entries, and they may distort analysis results if not handled properly. Using the duplicated() function in Pandas, 6,865 duplicate rows were detected in the dataset. These duplicate rows were removed to ensure that each transaction in the dataset was unique and that the analysis would not be biased by repeated data.During the exploration process, a new feature called Sales was created by multiplying the Quantity purchased by the Price of each product. This new variable represents the total value of each transaction and provides a more meaningful metric for analyzing sales performance. Creating new variables from existing data is a common feature engineering technique used in data mining to generate more informative insights.After completing the data cleaning and preparation steps, the dataset was reduced to 480,590 records and 9 attributes, including the newly created Sales feature. Several exploratory data analysis techniques and visualizations were then used to understand the distribution of sales, identify outliers, and explore relationships between variables. These analyses provided valuable insights into customer purchasing behavior and product demand patterns. One of the main challenges encountered during the process was handling the large size of the dataset and addressing the significant number of missing values in the Customer ID column. These challenges were addressed through efficient data cleaning techniques and careful preprocessing steps, ensuring that the final dataset was reliable and suitable for further data mining tasks in the upcoming stages of the project.
